Design log A location to host our data products

Data published: 26 June 2025

Version: Minimum Viable Product (MVP)

Why we did this work

Three of our data products (eDEN, eOPS and ePACT2) are currently hosted on Oracle. As of February 2026 the contract with Oracle is going to be terminated, this means we need to have a new location to host the data products.

The Data Hub was an assumed solution from consultancy work prior to user centred design involvement and an agreed approach to creating a centralised location for all things data, meaning users could access information, support, training and data products.

The initial version of the Data Hub is to have a location to host the data products post February 2026 and gradually introduce other areas of data iteratively.

What our ideas were

As our main goal is to have an MVP version of the Data Hub which would include accessing the three data products, and for users to be able to continue doing tasks they currently do when accessing the products.

Unfortunately, due to time restrictions the project did not have a Discovery phase and we were tasked to dive straight into Alpha, this was due to business requrements already being in place.

As the agreed MVP roadmap was to transition the Oracle version to the Data Hub version we have started with a lift and shift of the products while changing the UI to meet NHS Standards and following the NHS Toolkit, and iteratively introduce design changes to how a dashboard is displayed, how a user accesses the products, how the user accesses training and support or any news on the products, although these have been based on assumptions we are now taking this to usability testing to verify our assumptions with the user.

Prototype

The Prototype was split into 3 versions for MVP to iteratively introduce designs. We started with the basics for version 1, a user being able to view a list of products, sign into, request access, access a product, access reports and dashboards and existing training material, this would allow users to continue work using our data products.

Version 2 introduced access to a news page, landing pages for each data product and navigating from that page to a dashboard or report and slightly improved navigation to training materials and support.

For version 3 we focused on creating a better homepage which still just listed the data products but also introduced featured links, the future idea of the homepage is to streamline navigating between different areas of data as we introduce them. We also introduced basic filtering to product news, a new journey for users to create a data hub account and sign into an existing one along with a dedicated account space where users could see their own stored data analysis and folders.

How we tested them and what we found

UR IN PROGRESS

User research is starting on Monday 30 June 2025, this will be usability testing of the MVP Version 3 prototype. Usability testing will be task based, providing the users with 3 different tasks, those being related to account creation and sign in, navigating categories specifically product news and filtering and also accessing the data products and navigating them.

Findings

All findings will be provided here on completion of usability testing.

What we will do next

Depending on the outcome of usability testing we will take on the findings we have and any iterations needed will be prioritised and added to the roadmap.
